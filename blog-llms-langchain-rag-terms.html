LLMs, LangChain, RAG... Why So Many Terms?

A simple way to place common AI terms in one workflow map.

Back to All Posts

Published on April 6, 2026

One thing I have realized while trying to learn AI is that the hardest part at the start is not always the concepts themselves. It is the words.

Everywhere I look, there is a new term: LLMs, RAG, LangChain, embeddings, vector databases, agents. After a point it starts sounding like people are speaking in abbreviations on purpose.

For a while, I genuinely could not tell whether these were all separate things, whether they were competing tools, or whether they were just different names for the same idea. So I wanted to write this in the simplest way I can, mostly for myself, but maybe also for anyone else who has had the same confusion.

So, Is All Of It Just AI?

Yes, but also not in the way people casually say it. It is all part of the AI world, but these terms belong to different parts of the stack.

Some words refer to the model itself. Some refer to ways of making the model more useful. Some are frameworks for building applications around the model. Some are infrastructure pieces in the background.

Once I started looking at it that way, it stopped feeling like one giant mess of jargon.

The First Thing To Understand Is The LLM

At the center of most of this is the LLM. That is the actual language model. It is the part generating answers, summarizing text, writing code, explaining things, and responding to prompts.

So when people talk about AI applications, the LLM is usually the core engine underneath. That part is fairly straightforward.

Where it gets confusing is everything built around it.

Then Comes RAG

RAG was one of those terms that sounded more complicated than it turned out to be. It means the model is not answering purely from what it already knows. The system first pulls in relevant outside information, then asks the model to answer using that.

So instead of asking and hoping the model knows, the flow becomes:

  • Find the relevant information first
  • Give that information to the model
  • Let the model answer

This makes sense in real life because you often want answers based on actual documents, current notes, internal knowledge, or recent updates, not only training-time memory.

Embeddings Took Me A While To Understand

An embedding is basically a way of turning text into numbers so meaning can be compared. Not fancy, but it helped me.

If two sentences mean similar things, even with different words, embeddings help the system realize they are related. That matters because retrieval should still work when wording does not match exactly.

So embeddings are part of why retrieval can work well.

And Then There Are Vector Databases

Once text is converted into embeddings, you need somewhere to store and search them efficiently. That is where vector databases come in.

It is not some magical AI brain. It is a storage and search layer for meaning-based representations.

A simple RAG pipeline often looks like this:

  • Take documents
  • Split into smaller parts
  • Turn parts into embeddings
  • Store in a searchable vector index
  • When a question comes in, retrieve the most relevant parts
  • Pass those parts to the LLM

LangChain Confused Me For A Different Reason

For a long time, I was not sure what LangChain was supposed to be. I thought it might be another model. It is not.

LangChain is more like a framework for building around LLMs. It helps connect prompts, retrieval, memory, tools, and workflows.

The way I think of it now:

  • The LLM is the engine
  • LangChain helps wire the rest of the machine

It is not the intelligence itself. It is part of the application layer around the intelligence.

Agents Sound More Dramatic Than They Need To

The word agent sounds futuristic. Sometimes it is interesting. Sometimes it is just branding.

At a practical level, an agent is usually a model system that can do more than reply once. It can decide to use tools, fetch information, run steps in sequence, and work toward an outcome.

So instead of one direct answer, it might search, call a tool, run a function, and then return a result. Helpful, but the term is often stretched.

A simple way to hold it: agents are LLM systems that can act, not only respond.

APIs Are Still Quietly Underneath All Of This

A lot of new AI language makes it sound like everything has changed. But APIs are still APIs. They still connect applications to models, tools, databases, and external systems.

So while people discuss new AI concepts, much of the plumbing remains familiar. That is comforting, because not everything is brand new. Some of this is classic software ideas meeting new model capabilities.

What Helped Me Most Was Seeing The Workflow

I needed flow, not isolated definitions.

  • A user asks something
  • The application receives the question
  • If needed, it retrieves relevant information
  • That context is passed to the LLM
  • The LLM generates the response
  • The application shows the result

Some systems are simpler, some are more complex. But once I saw the flow, the vocabulary stopped feeling random. Each term had a place.

Why AI Feels Overwhelming At The Beginning

People often talk about every layer at once: models, frameworks, retrieval, GPUs, agents, evaluation. If you are new, it arrives as one giant cloud of terminology.

It gets easier once you separate layers:

  • The LLM is the model
  • RAG is how the model gets outside knowledge
  • Embeddings help match meaning
  • Vector databases store and search those representations
  • LangChain helps orchestrate workflows
  • Agents add tool use and multi-step behavior
  • APIs connect everything together

Not simple-simple, but at least understandable.

The Biggest Shift For Me

The most useful AI products are not just a model. They are systems built around a model.

That changes how to learn this space: not by memorizing terms in isolation, but by understanding where each one fits.

Once you know each role, the whole thing looks less like noise and more like architecture.

Final Thought

A lot of AI feels harder than it is because of how it gets discussed. Too many terms, too much jargon, and too many assumptions that everyone already knows the differences.

The more I read, the more I feel this gets clearer once you stop asking what each word means in isolation and start asking where it fits.

That question has helped me a lot. I am still learning, but at least the map in my head is less messy than it was a few weeks ago.
